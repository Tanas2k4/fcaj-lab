This page records the entire learning and internship journey on Amazon Web Services (AWS) cloud computing over a 12-week period (approximately 3 months). Each week focuses on combining theoretical research with in-depth hands-on labs to master key AWS services.

The weekly work logs are summarized as follows:

* **Week 1:** [Getting familiar with AWS, Budgets, IAM, CLI, and VPC](1.1-Week1/) - Set up an AWS account, monitor budget alerts, manage access controls with IAM, configure the AWS CLI locally, and deploy a basic VPC networking structure.
* **Week 2:** [Secure Connection to Private Instances and DNS Resolution](1.2-Week2/) - Practice secure connectivity methods (Instance Connect Endpoint, RDGW) to servers in private subnets and perform internal DNS resolution testing.
* **Week 3:** [Advanced EC2, Auto Scaling, and CloudFront](1.3-Week3/) - Deep dive into EC2 Auto Scaling, complete Module 3 hands-on labs (Lab 13, 24, 57), and explore Amazon CloudFront CDN configuration.
* **Week 4:** [AWS Storage Services and Amazon S3](1.4-Week4/) - Overview of AWS Storage services, in-depth study of Amazon S3 (static website hosting, CORS), and practice related labs.
* **Week 5:** [Advanced Security and Identity Management](1.5-Week5/) - Study advanced IAM, AWS Cognito, AWS Organizations, IAM Identity Center, Security Groups, and AWS Security Hub.
* **Week 6:** [Technology Research and Project Planning](1.6-Week6/) - Research technologies to be used in the project, estimate costs with AWS Pricing Calculator, and plan task assignment for team members.
* **Week 7:** [Application Deployment Trial on EC2 and RDS](1.7-Week7/) - Review EC2 and RDS configurations, push source code to EC2 instance, and set up the environment for successful application run.
* **Week 8:** [RDS Integration, CloudFront, and AWS Blog Research](1.8-Week8/) - Configure source code connection with RDS, learn about CloudFront, and research technical blogs on Amazon Web Services.
* **Week 9:** [Frontend CI Setup and Backend Local Testing](1.9-Week9/) - Research and set up automated CI for Frontend, restructure the project, and build a local test environment for Backend matching Production.
* **Week 10:** [Feature Completion, S3 Integration, and SES Research](1.10-Week10/) - Resolve merge conflicts, complete project features, improve UI/UX, integrate AWS S3, and research AWS SES for email notification.
* **Week 11:** [System-wide Testing and Migration to Private Subnet](1.11-Week11/) - Resolve branch conflicts, move server to private subnet, conduct comprehensive system testing, and monitor all completed features.
* **Week 12:** [Final Review and Internship Evaluation](1.12-Week12/) - Consolidate all learned skills, finalize the internship report, and evaluate achievements from the internship.
